# RAPS Baseline Comparison: Simulator Validation

This directory contains the infrastructure for comparing RAPS's analytical network
model against packet-level and flow-level simulators on identical workloads.

## Motivation

Reviewer concern: *"There are no accuracy/fidelity comparisons showing that RAPS produces
outputs that agree with packet-level simulators on the same workload."*

## Common Experimental Configuration

### Topology 1: Dragonfly (Frontier-like)
- **Groups:** 9 · **Routers/group:** 4 · **Hosts/router:** 2 · **Total hosts:** 72
- **Link bandwidth:** 25 GB/s · **Link latency:** 100 ns
- **Routing:** minimal (deterministic shortest path)
- See `configs/dragonfly_ref.yaml` for full spec

### Topology 2: Fat-tree (Lassen-like)
- **k=4** fat-tree · **Total hosts:** 16 · **3-level** (edge/agg/core)
- **Link bandwidth:** 12.5 GB/s · **Link latency:** 100 ns
- **Routing:** ECMP
- See `configs/fattree_ref.yaml` for full spec

### Traffic Pattern
- All-to-all (uniform random permutation)
- **Injection rates ρ ∈ {0.05, 0.10, 0.20, 0.30, 0.40, 0.50, 0.60, 0.70, 0.80}**

### Output Metrics
- **mean_utilization**: mean link utilization across all links
- **max_utilization**: hottest link utilization (= ρ by construction in RAPS)
- **stall_ratio**: (observed_latency − zero_load_latency) / zero_load_latency
- **slowdown**: observed_latency / zero_load_latency
- **avg_latency_ns**: end-to-end packet/flow latency

## Metric Definitions

### Link Utilization
```
utilization(link) = bytes_transferred / (bandwidth × time_window)
mean_utilization  = mean over all links
max_utilization   = max over all links (= ρ by construction in RAPS)
```

### Stall Ratio
```
stall_ratio = (observed_latency - zero_load_latency) / zero_load_latency
            ≈ ρ² / (2·(1-ρ))    [M/D/1 queue theory]
```

### RAPS M/D/1 Model
RAPS uses the M/D/1 queue formula:
```
slowdown = 1 + ρ² / (2·(1-ρ))
stall_ratio = ρ² / (2·(1-ρ))
```
This is derived from queueing theory for a deterministic service time queue at
utilization ρ. RAPS applies this at the network tick level when congestion is detected.

Zero-load latencies (used for stall_ratio computation):
- Dragonfly minimal: 3 hops × 100 ns + 50 ns injection = 350 ns
- Fat-tree ECMP:     4 hops × 100 ns + 50 ns injection = 450 ns
